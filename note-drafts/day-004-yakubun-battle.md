# 約分RPGバトル──算数の「つまずき」をゲームで突破する

## 1. なぜ作ったか──授業での約分の難しさ

約分は小学校算数の中でも、子どもたちが特につまずきやすい単元のひとつです。「分子と分母を同じ数で割る」という手順は理屈で伝わっても、**スピードと正確さを同時に鍛える練習機会**がなかなかつくれないのが長年の悩みでした。

ドリル形式だと「やらされ感」が出る。タイムアタックだと焦るだけ。

「対人戦にすれば、競争心が勉強の燃料になるのでは？」──それがこのゲームの出発点です。

## 2. ゲームの特徴

### タブレット1台で対面2人同時プレイ

画面を机の上に置き、向かい合った2人がそれぞれ自分側のテンキーを操作します。プレイヤー1のエリアは**180度回転**しているので、両者が自分向きに画面を読めるのがポイント。アプリのインストール不要、ブラウザで開くだけです。

### 8体のキャラクターから選ぶ楽しさ

キャラクター選択も同時進行です。画面を挟んで2人が向かい合い、それぞれ自分側から好きなキャラを選ぶと、3・2・1のカウントダウンが始まってそのままバトルへ突入します。「次はこっち選ぼう」という会話が自然に生まれます。

| キャラ | HP | 攻撃 | 必殺技 |
|--------|-----|------|--------|
| 🛡️ 騎士 | 150 | 20 | 二連撃（ダメージ2倍） |
| 🔮 魔法使い | 100 | 35 | 大魔法（HP全回復） |
| 🥷 忍者 | 120 | 25 | 影分身（相手に追加問題） |
| 👹 モンスター | 130 | 30 | 大暴れ（相手HP半減） |
| 🐉 竜 | 160 | 18 | 炎ブレス（固定60ダメージ） |
| 👼 天使 | 110 | 28 | 聖なる光（自分HP半回復） |
| 🏴‍☠️ 海賊 | 140 | 25 | 強奪（相手HP30を奪う） |
| 😈 魔王 | 90 | 45 | 三連撃（ダメージ3倍） |

「HPの高いキャラと攻撃力の高いキャラ、どっちが強い？」という議論だけで休み時間が終わります（笑）。

### テンキーの入力順が「約分の思考順」

テンキーの入力順は**分母→「／」→分子→答え合わせ**です。最初に「何で割れそう？」と分母を決め、次に分子を確認する、という思考の流れがそのまま操作になっています。算数の授業で「まず分母を見なさい」と伝えることが多いので、この順序にこだわりました。

### 問題はランダム生成（公約数2〜9）

公約数2〜9の約分問題をランダムに生成します。答えは必ず既約分数になるようロジックを組んでいるので、子どもが「これ、もっと約分できる？」と悩む場面がありません。公約数が大きいほど難易度が上がり、9なら `9/81` のような歯ごたえある問題も出てきます。

問題は**各プレイヤーのパネル内**にそれぞれ表示されます。P1側は回転しているパネルの中に問題があるので、どちらからでも正位置で読めます。

## 3. Claude Codeへの指示で工夫したこと

### 4部署に分けて順番に作業させた

```
【デザイン部】→【コーディング部】→【QA部】→【note記事部】
```

ロールを分けることで、コーディング部が「何を作ればいいか」に迷わず、QA部が「何を確認すればいいか」を明確に持てました。特に**QA部に具体的なチェックリストを渡す**のが効果的で、自分でバグを発見・修正してから記事部に渡す流れが自然に作れました。

### バグ修正のやりとりで気づいたこと

最初の実装で見つかったバグは4つ。

1. 必殺技フラグがゲーム再起動時にリセットされない
2. 忍者スキルの追加問題数の設定値がずれていた
3. 両プレイヤーが同時に正解するとラウンドが2回進む
4. ゲーム終了後も入力できてしまう

これをQA部への指示としてそのまま渡したら、コードを読んで原因まで特定して直してくれました。「バグ修正して」より「こういう動作になるはずが、こうなってしまう」と具体的に書くほうが精度が上がると実感しました。

### 追加要望も自然に反映できた

完成後に「分母から入力したい」「問題を両側から見えるように」「キャラを8体に」と追加要望を出すたびに、ロジックへの影響を確認しながら対応してもらえました。特に入力順の変更（分子→分母 から 分母→分子）は、`phase`の初期値・フェーズ遷移・表示ロジック・ラベルと複数箇所に影響するのですが、漏れなく一括で修正されていました。

## 4. 次の課題

- **CPUモード**：一人で練習できるコンピュータ対戦があると授業外でも使えそう
- **スコア表示**：ラウンド数・正解数を勝敗画面に出したい
- **難易度選択**：「かんたん（公約数2〜4）」「ふつう」「むずかしい」と選べると学年対応できる
- **通分・大小比較への拡張**：分数の学習はまだ続くので、同じUIで応用できるはず

タブレットを机に置いて2人で「せーの」で解く瞬間、子どもたちがどんな顔をするか、早く試してみたいです。
